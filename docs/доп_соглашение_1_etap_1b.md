## 1. Предмет дополнительного соглашения / Subject of Supplementary Agreement

Настоящее Дополнительное соглашение определяет работы по разработке и внедрению дополнительных функций платформы, которые логически продолжают Этап 1 (MVP) Основного договора и целесообразно должны быть завершены перед переходом к Этапам 2 и 3 Основного договора. 1. **Упрощенные личные кабинеты** для заказчиков, администраторов и исполнителей (мастеров)

3. **Расширенная классификация работ** с новой системой кодировки (UTM-style)

## 2. Описание работ / Description of Works

### 2.1. Упрощенные личные кабинеты / Simplified User Dashboards

Исполнитель разработает и внедрит персонализированные интерфейсы для трех типов пользователей платформы:

#### 2.1.1. Личный кабинет заказчика / Client Dashboard
- **Аутентификация и регистрация:** Интеграция с существующей системой аутентификации платформы
- **Создание заявок:** Упрощенная форма для подачи заявок на обслуживание с предзаполненными данными из профиля
- **Отслеживание статуса:** Реал-тайм мониторинг статуса заявок (новая, в обработке, выполнена и т.д.)
- **История заявок:** Просмотр всех предыдущих заявок с детальной информацией
- **Управление профилем:** Редактирование контактных данных, адресов и предпочтений
- **Уведомления:** Push-уведомления и email-оповещения о изменениях статуса заявок

#### 2.1.2. Личный кабинет администратора / Admin Dashboard
- **Управление пользователями:** Просмотр и управление профилями клиентов и мастеров
- **Редактирование заявок:** Редактирование заявок на обслуживание с предзаполненными данными
- **Панель управления заявками:** Фильтрация, сортировка и массовые операции с заявками
- **Аналитика и отчеты:** Базовые метрики по заявкам, мастерам и клиентам
- **Управление контентом:** Редактирование справочной информации и настроек платформы


#### 2.1.3. Личный кабинет мастера (исполнителя) / Master Dashboard
- **Профиль специалиста:** Управление специализацией, опытом и рейтингом
- **Управление заявками:** Просмотр назначенных заявок, обновление статуса
- **Финансовая информация:** Просмотр выполненных работ и заработка
- **Отзывы и рейтинг:** Просмотр отзывов клиентов и текущего рейтинга

#### 2.1.4. Технические требования / Technical Requirements
- **Адаптивный дизайн:** Полная поддержка мобильных устройств и планшетов
- **Интеграция:** Полная интеграция с существующей базой данных и API платформы

### 2.2. Модуль управления контрактами / Contract Management Module

Исполнитель разработает систему для создания и управления контрактами с автоматической генерацией n8n workflow'ов:

#### 2.2.1. Форма создания контрактов / Contract Creation Form
- **Ввод данных контракта:** Поля для всех необходимых параметров контракта
- **Валидация данных:** Проверка корректности введенных данных в реальном времени
- **Шаблоны контрактов:** Предустановленные шаблоны для разных типов контрактов

#### 2.2.2. Интеграция с n8n / n8n Integration
- **API подключение:** Интеграция с n8n API для создания и управления workflow'ами
- **Автоматическая генерация:** Создание workflow'ов на основе данных контракта (+ доступные credentials)
- **Webhook endpoints:** Генерация уникальных webhook URL для каждого контракта
- **Мониторинг:** Отслеживание статуса workflow'ов и их выполнение

#### 2.2.3. База данных контрактов / Contract Database
- **Таблица контрактов:** Новая таблица для хранения данных контрактов
- **Связь с пользователями:** Привязка контрактов к клиентам и мастерам
- **История изменений:** Логирование всех изменений в контрактах
- **Поиск и фильтрация:** Полнотекстовый поиск и фильтры по различным параметрам

#### 2.2.4. Технические требования / Technical Requirements
- **Масштабируемость:** Поддержка роста количества контрактов
- **Безопасность:** Шифрование чувствительных данных контрактов

### 2.3. Расширенная классификация работ / Extended Work Classification

Исполнитель расширит систему классификации работ новыми категориями и подкатегориями:

#### 2.3.1. Новая структура классификации / New Classification Structure
- **Категории:** Garten & Außenanlagen, Reinigung & Haushalt, Handwerker Hobby, Handwerk – Fachlich, Service & Betreuung, Unterstützung & Alltag, Technik & Digital
- **Подкатегории:** Детальная разбивка каждой категории на подтипы работ

#### 2.3.2. Система кодировки (UTM-style) / UTM-Style Coding System
- **Уникальные коды:** Генерация уникальных кодов для каждой работы
- **Параметры кодировки:** Включение типа работы, региона, срочности и др.
- **Отслеживание источников:** Аналитика по источникам заявок

#### 2.3.3. Миграция данных / Data Migration
- **Обновление базы данных:** Добавление новых полей и таблиц
- **Перенос существующих данных:** Миграция текущих заявок на новую структуру
- **Тестирование:** Проверка корректности миграции данных

#### 2.3.4. Технические требования / Technical Requirements
- **Обратная совместимость:** Поддержка старых заявок с новой классификацией
- **Производительность:** Быстрый поиск и фильтрация по новой структуре
- **Гибкость:** Легкое добавление новых категорий без изменения кода
