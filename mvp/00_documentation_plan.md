# План документации MVP AssetCare24

**Философия:** "Just Enough Documentation" (Ровно столько, сколько нужно для реализации).
**Принцип:** Документ → Реализация → Тест → Обновление документа.

## Структура документации (Слои)

Мы разделим документацию на 4 понятных слоя. Каждый слой отвечает на конкретный вопрос разработчика.

### 1. Слой Бизнес-логики (Потоки)
*Отвечает на вопрос: "Что происходит?"*
*   **01_main_flow.md** — Главный поток данных (Happy Path: Заявка → Результат).
*   **02_user_roles.md** — Кто есть кто (Клиент vs Мастер vs Админ) и их права в MVP.

### 2. Слой Данных (Память)
*Отвечает на вопрос: "Где и как храним?"*
*   **03_database_schema_mvp.md** — Упрощенная схема БД (Supabase). Только таблицы, нужные для Главного потока.
*   **04_storage_structure.md** — Структура папок в MinIO (S3) для фото/видео.

### 3. Слой Автоматизации (Мозги)
*Отвечает на вопрос: "Как обрабатываем?"*
*   **05_n8n_workflows.md** — Описание сценариев n8n.
    *   *Scenario A:* Входящая заявка (WhatsApp/Web) → AI разбор → БД.
    *   *Scenario B:* Маршрутизация → Уведомление мастера.
    *   *Scenario C:* Завершение работы → Сбор отзыва.

### 4. Слой Интерфейсов (Витрина)
*Отвечает на вопрос: "Что видит пользователь?"*
*   **06_whatsapp_bot_logic.md** — Скрипты диалогов (тексты и кнопки) для Twilio.
*   **07_website_mvp.md** — ТЗ на упрощение текущего сайта (только посадочная + форма заявки).

---

## План действий (Итерации)

1.  **Итерация 1: Фундамент (Текущий этап)**
    *   Утвердить `01_main_flow`.
    *   Создать `03_database_schema_mvp` (SQL для Supabase).
    *   Настроить бакеты MinIO по `04_storage_structure`.

2.  **Итерация 2: "Труба" (Pipeline)**
    *   Настроить n8n workflow приема заявки (Webhook от Twilio/Site).
    *   Запись в БД.
    *   Сохранение файла в MinIO.

3.  **Итерация 3: Коммуникация**
    *   Логика оповещения мастера (Telegram группа или личный WhatsApp).
    *   Обратная связь от мастера (кнопки "Принял", "Сделал").

4.  **Итерация 4: WhatsApp-first интеграция** ✅ *ВЫПОЛНЕНО*
    *   Полный переход на WhatsApp-first подход.
    *   Удаление форм заявок клиентов.
    *   Прямые ссылки на WhatsApp из всех точек контакта.
